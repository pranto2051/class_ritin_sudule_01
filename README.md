# Class Rotin - University Class Routine Manager

A web app to manage university class routine data (subjects, teachers, rooms, days, time slots, and schedules) with role-based authentication.

## What This Project Includes

- Authentication with Supabase Auth
- Role-based access (`super_admin`, `admin`, `teacher`, `student`)
- Admin CRUD pages for:
	- Subjects
	- Teachers
	- Rooms
	- Days
	- Time Slots
	- Schedule Entries
- Onboarding and dashboard flow
- Built-in demo mode (works without Supabase env vars)

## Tech Stack

- TanStack Start + React + TypeScript
- Vite
- Supabase
- Tailwind CSS + Radix UI components

## Prerequisites

Install these first:

- Node.js 20+ (or Bun)
- Package manager:
	- Bun (recommended in this repo), or
	- npm

Optional (only if you want local Supabase DB/auth):

- Supabase CLI
- Docker Desktop (running)

## 1. Install Dependencies

From project root:

```bash
bun install
```

Or with npm:

```bash
npm install
```

## 2. Environment Setup

This app can run in two modes:

### Mode A: Demo Mode (Fastest)

No `.env` file is required.

If `VITE_SUPABASE_URL` and `VITE_SUPABASE_PUBLISHABLE_KEY` are missing, the app automatically uses an in-browser demo backend with seeded demo data and demo logins.

### Mode B: Real Supabase Mode

Create a `.env` file in project root and add:

```env
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_PUBLISHABLE_KEY=your_supabase_anon_or_publishable_key
```

Then restart the dev server.

## 3. Run Locally

Start dev server:

```bash
bun run dev
```

Or:

```bash
npm run dev
```

App URL:

- http://localhost:8080

## 4. Default Login Information

### Demo Accounts (Demo Mode)

Use these credentials on `/login`:

| Role | Email | Password |
| --- | --- | --- |
| Super Admin | admin@demo.edu | demoadmin |
| Admin | admin2@demo.edu | demoadmin |
| Teacher | teacher@demo.edu | demoteacher |
| Student | student@demo.edu | demostudent |

Notes:

- On the login page, clicking a demo account auto-fills credentials.
- These accounts are guaranteed in demo mode.

### Real Supabase Role Behavior

- The first registered user becomes `super_admin` automatically.
- Later users default to `student` unless promoted.
- Promotion is controlled by records in `public.user_roles`.

## 5. Optional: Local Supabase Development

If using Supabase CLI with the migration files in `supabase/migrations/`:

```bash
supabase start
supabase db reset
```

After reset, ensure your `.env` points to the local project URL/key generated by Supabase CLI.

## Troubleshooting

- If login fails in demo mode, use the exact credentials listed above.
- If app shows Supabase env warning, either:
	- add valid `VITE_SUPABASE_*` values, or
	- continue with demo mode.
- If local Supabase commands fail, verify Docker is running.

## Project Structure (Important Paths)

- `src/routes/` - Route pages
- `src/components/` - Reusable UI/components
- `src/integrations/supabase/client.ts` - Supabase client and demo fallback logic
- `supabase/migrations/` - SQL schema and role logic
